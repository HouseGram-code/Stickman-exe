### ü§ù Contributing & Rules
Want to make your own changes or add new features? We welcome contributions! Please read the rules before submitting a Pull Request.

<details>
<summary><b>üìú Click to view the 5 Golden Rules of Contribution</b></summary>
<br>

1. **Keep the Horror Vibe:** Any new assets, sounds, or mechanics must fit the creepy, unsettling atmosphere of the original game.
2. **No External Libraries for Core Mechanics:** The 2.5D engine is built purely on CSS and React state. Do not add heavy 3D libraries (like Three.js) unless discussing it in an issue first.
3. **Mobile First:** Ensure any new UI elements or controls work flawlessly on touch devices.
4. **Clean Code:** Use TypeScript strictly. Avoid `any`. Keep components modular.
5. **Test Your Scares:** Make sure jumpscares trigger correctly and audio contexts are handled properly across different browsers.
